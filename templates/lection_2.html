Графы в Информатике для ЕГЭ

Введение в теорию графов

Граф — это структура данных, состоящая из вершин (узлов) и ребер (связей между вершинами). Графы используются для моделирования отношений между объектами.

Графы могут быть ориентированными (где связи имеют направление) и неориентированными (где связи не имеют направления).

Для ЕГЭ по информатике важно понимать основные понятия и уметь работать с графами, представленными в разных формах.

Основные понятия

  • Вершина (узел): Представляет собой объект в графе.
  • Ребро: Связь между двумя вершинами.
  • Смежные вершины: Вершины, соединенные ребром.
  • Путь: Последовательность вершин, соединенных ребрами.
  • Цикл: Путь, начинающийся и заканчивающийся в одной и той же вершине.
  • Степень вершины: Количество ребер, инцидентных вершине (для неориентированных графов).
  • Полустепень захода/исхода: Количество входящих/исходящих ребер (для ориентированных графов)
  • Взвешенный граф: Граф, где каждому ребру присвоено число (вес, стоимость).

Способы представления графов

Графы можно представлять разными способами:

  1. Матрица смежности: Двумерный массив, где элемент (i, j) равен 1, если существует ребро между вершинами i и j, и 0 иначе.
  2. Список смежности: Список, где для каждой вершины хранится список смежных вершин.
  3. Матрица инцидентности: Двумерный массив, где строки соответствуют вершинам, а столбцы ребрам.

Пример матрицы смежности

В этом примере, если есть 1 на пересечении строки и столбца, то вершины связаны между собой. Например, вершины A и B связаны, а вершины A и D нет.

Пример списка смежности

A: [B, C]
B: [A, C, D]
C: [A, B, D]
D: [B, C]

Этот пример показывает, что вершина A смежна с вершинами B и C и т.д.

Задачи на графы в ЕГЭ

В ЕГЭ по информатике встречаются различные задачи на графы, которые требуют:

  • Определения количества путей между вершинами.
  • Нахождения кратчайшего пути между вершинами.
  • Анализа связности графов.
  • Работы с взвешенными графами.

Типичные задания

1. Определение количества путей. В таких задачах нужно посчитать количество путей определенной длины, либо из одной точки в другую. Обычно используется обход графа, который мы сейчас обсудим.

2. Определение кратчайшего пути. Для таких задач существует ряд специальных алгоритмов, однако на ЕГЭ можно обойтись ручным подсчетом.

Обход графа

Существует два основных способа обхода графа:

  • Поиск в ширину (BFS): Изучает граф по уровням, начиная от начальной вершины. Использует очередь.
  • Поиск в глубину (DFS): Исследует одну ветвь графа до конца, затем возвращается и исследует другую ветвь. Использует стек (рекурсию).

Пример DFS

def dfs(graph, vertex, visited):
    visited[vertex] = True
    print(vertex, end=" ")

    for neighbor in graph[vertex]:
        if not visited[neighbor]:
            dfs(graph, neighbor, visited)

# Пример графа в виде списка смежности
graph = {
    'A': ['B', 'C'],
    'B': ['A', 'D', 'E'],
    'C': ['A', 'F'],
    'D': ['B'],
    'E': ['B', 'F'],
    'F': ['C', 'E']
}

visited = {vertex: False for vertex in graph}
dfs(graph, 'A', visited) # Запуск DFS из вершины A

В этом примере, функция dfs проходится по вершинам и рекурсивно вызывает саму себя для необработанных вершин.

Пример BFS

from collections import deque

def bfs(graph, start_vertex):
    visited = {vertex: False for vertex in graph}
    queue = deque([start_vertex])
    visited[start_vertex] = True

    while queue:
        vertex = queue.popleft()
        print(vertex, end=" ")

        for neighbor in graph[vertex]:
            if not visited[neighbor]:
                visited[neighbor] = True
                queue.append(neighbor)

graph = {
    'A': ['B', 'C'],
    'B': ['A', 'D', 'E'],
    'C': ['A', 'F'],
    'D': ['B'],
    'E': ['B', 'F'],
    'F': ['C', 'E']
}

bfs(graph, 'A') # Запуск BFS из вершины A

В этом примере, функция bfs использует очередь и проходит граф по уровням, сначала посещая всех соседей стартовой вершины, а потом их соседей и т.д.

Важные советы для ЕГЭ

  • Обязательно переводите условия задачи в конкретное представление графа.
  • Сначала делайте простые наброски на бумаге.
  • Понимание алгоритмов обхода графа (DFS, BFS) может помочь в решении сложных задач.
  • Проверяйте ваши решения на небольших примерах.
